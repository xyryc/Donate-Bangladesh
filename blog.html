Important FAQ

1. What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface that represents the structure of an HTML or XML document as a tree of objects. Each element, attribute, and piece of content in a document becomes a node in this tree, which can be accessed, modified, and manipulated using languages like JavaScript. The DOM allows developers to dynamically change the content, structure, and style of a webpage.

2. How do you select an element from the DOM?

To select an element from the DOM, JavaScript provides various methods. Some common methods include: getElementById(): Selects an element by its unique ID. getElementsByClassName(): Selects all elements with a given class name. getElementsByTagName(): Selects all elements with a specific tag. querySelector(): Selects the first element that matches a CSS selector. querySelectorAll(): Selects all elements that match a CSS selector.

3. What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where instead of attaching event listeners to individual child elements, you attach a single event listener to a parent element. This event listener can detect events triggered by its child elements using the event's propagation (bubbling) process. It is useful because: It reduces memory usage by avoiding multiple event listeners. It simplifies event handling when dealing with dynamically added elements, as the parent can manage events for its future children.

4. How do you manipulate an element's attributes and styles using the DOM?

You can manipulate an element's attributes using methods like: setAttribute(): Sets a new value for an attribute (e.g., setAttribute("src", "image.jpg")). getAttribute(): Retrieves the value of an attribute. removeAttribute(): Removes an attribute from an element.
